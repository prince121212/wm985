# 文明知识库小程序 MVP 原型

这是一个用于决策的小程序端最小产品原型，不是正式微信小程序代码。

## 覆盖页面

1. 首页：搜索、分类入口、热门资源
2. 资源库：分类筛选、资源列表
3. 资源详情：资源信息、标签、收藏、访问资源
4. 上传资源：标题、链接、描述、分类、标签、提交审核
5. 我的：积分、收藏、上传、反馈入口

## MVP 边界

- 不做管理后台，继续使用网站后台。
- 小程序端暂不做付费访问，第一版只展示免费资源。
- 不做真实文件上传，先支持外部链接提交。
- 每个页面都有分享入口；好友通过分享进入小程序即算分享成功，双方各得 20 积分。
- 后端优先复用现有 Next.js API，必要时新增 `/api/mp/*` 和分享归因接口。


## 分享积分机制草案

- 分享人点击页面分享按钮，生成带 `share_user`、`scene`、`target_id` 的小程序路径。
- 好友首次通过该路径进入小程序，即记录一次有效分享。
- 有效分享成立后，分享人和好友各获得 20 积分。
- 防刷建议：同一好友对同一分享人只奖励一次；同设备/同 openid 不重复奖励；每日分享奖励可设置上限。


## 资源访问策略

- 小程序端按钮文案仍保持「访问资源」。
- 点击「访问资源」不尝试 WebView，也不跳外链，统一复制原始资源链接。
- 复制成功后提示：资源链接已复制，请在浏览器或对应 App 中打开。
- 这样可以兼容网盘、第三方资源站、工具网站和其他任意外链。
